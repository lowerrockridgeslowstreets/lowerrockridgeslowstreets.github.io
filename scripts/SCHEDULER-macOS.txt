Formspree sync on macOS (cron)
===============================

Cron could not open the script under ~/Documents (Operation not permitted). Fix:

1) Copy the script OUT of Documents (repo stays where it is):

   mkdir -p ~/bin
   cp /Users/USER/Documents/GitHub/rockridge-slow-neighborhood-staging/scripts/sync_supporter_count_formspree.py ~/bin/
   chmod +x ~/bin/sync_supporter_count_formspree.py

   After "git pull", re-copy ~/bin from scripts/ if the script changed.

2) Put API secrets where cron can read them (NOT only under Documents):

   mkdir -p ~/.config
   cp private/formspree-sync.env ~/.config/rockridge-formspree.env
   (or create ~/.config/rockridge-formspree.env with FORMSPREE_API_KEY=...)

3) Crontab — one line (remove any older */5 line that cd'd into the repo):

*/5 * * * * ROCKRIDGE_REPO=/Users/USER/Documents/GitHub/rockridge-slow-neighborhood-staging GIT_PUSH=1 /usr/bin/python3 /Users/USER/bin/sync_supporter_count_formspree.py >> /tmp/formspree-sync.log 2>&1

4) If "git push" still fails under cron, macOS is blocking git in ~/Documents:
   add /usr/sbin/cron to Full Disk Access (⌘⇧G → /usr/sbin → cron), OR move the repo.

Log: tail -f /tmp/formspree-sync.log

Optional: LaunchAgent plist in this folder if you prefer that over cron.
